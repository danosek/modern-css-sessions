Celá navigace je stylovaná jedním vnořeným blokem .nav-demo { … }

Základní nesting — flat CSS vs. nested CSS:

Flat CSS (tradiční)
.card { background: …; border-radius: …; }
.card__title { font-size: …; color: …; }
.card__body { padding: …; color: …; }
.card__footer { border-top: …; }
.card:hover { box-shadow: …; }
.card--featured { border-color: …; }
Selektor .card se opakuje → obtížnější přesun, rename
Nested CSS (nativní)
.card {
  background: …;
  border-radius: …;

  & .card__title { font-size: …; color: …; }
  & .card__body  { padding: …;  color: …; }
  & .card__footer { border-top: …; }
  &:hover { box-shadow: …; }
  &.card--featured { border-color: …; }
}
Vše v jednom bloku → přehlednost, snadný refactoring

Nesting bez & — element selektory uvnitř bloku ampersand nepotřebují:

Základní karta

Element h2 a p jsou psané bez ampersandu — stačí je vnořit přímo. Prohlédni si hover efekt.

footer · h2 a p bez &

Aktivní stav

Modifier &.no-amp-card--active stále vyžaduje &. Uvnitř pak opět h2 bez &.

footer · &.--active + h2 bez & 
.no-amp-card {
  border-radius: var(--radius-strong);
  padding: var(--base-3) var(--base-4);

  /* element selektory nepotřebují & — platí od Chrome 112+, FF 117+, Safari 17.2+ */
  h2 { font-size: var(--font-size-xl); color: var(--text-primary); }
  p  { color: var(--text-secondary); line-height: 1.6; }
  span { font-family: var(--font-stack-monospace); }

  /* třídy, pseudo-třídy a pseudo-elementy & vyžadují nadále */
  &:hover          { box-shadow: var(--elevation); }
  &.no-amp-card--active {
    border-color: var(--surface-brand-primary-strong);
    h2 { color: var(--text-brand-primary); }  /* opět bez & */
  }
}

Pravidlo: selektory začínající identifikátorem (element, *) implicitně dostanou & jako předponu — výsledek je .no-amp-card h2. Třídy (.foo), atributy ([type]) a pseudo-třídy (:hover) se bez & nepřipojí správně — prohlížeč je interpretuje jako & .foo (s mezerou = potomek), ne &.foo (bez mezery = modifier).

@media uvnitř pravidla — zmenši okno prohlížeče a sleduj změnu:

Responzivní nadpis

Tento blok mění layout, velikost písma i padding — vše definováno uvnitř jednoho CSS pravidla.

Desktop Mobil 
.fluid-hero {
  display: flex;
  flex-direction: row;       /* desktop: řádek */
  gap: var(--base-4);
  padding: var(--base-5);

  /* @media přímo uvnitř pravidla! */
  @media (max-width: 600px) {
    flex-direction: column;  /* mobil: sloupec */
    padding: var(--base-3);
  }

  & .fluid-hero__title {
    font-size: var(--font-size-xxl);

    @media (max-width: 600px) {
      font-size: var(--font-size-xl);  /* menší na mobilu */
    }
  }
}

CSS Nesting — klíčové vzory

/* 1. Základní nesting — & je odkaz na rodičovský selektor */
.nav {
  display: flex;
  gap: var(--base-2);

  & .nav__item { list-style: none; }   /* potomek */
  & .nav__link { text-decoration: none; color: var(--text-primary); }
  & .nav__link:hover { color: var(--text-brand-primary); } /* pseudo-třída */
}

/* 2. Modifier & state vše v jednom bloku */
.btn {
  padding: var(--base-h) var(--base-3);
  background: var(--surface-brand-primary-strong);

  &:hover        { filter: brightness(1.1); }
  &:disabled     { opacity: 0.4; cursor: not-allowed; }
  &:focus-visible { outline: 2px solid currentColor; outline-offset: 3px; }
  &.btn--ghost   { background: transparent; border: 1px solid currentColor; }
}

/* 3. @media vnořené přímo v pravidle */
.hero {
  font-size: var(--font-size-xxl);
  flex-direction: row;

  @media (max-width: 600px) {
    font-size: var(--font-size-xl);   /* bez opakování selektoru */
    flex-direction: column;
  }
}

/* 4. Nesting bez & — od Chrome 112+, Firefox 117+, Safari 17.2+ */
.card {
  border-radius: var(--radius);

  .card__title { font-weight: 600; }  /* implicitní & před potomkem */
  .card__body  { color: var(--text-secondary); }
}
