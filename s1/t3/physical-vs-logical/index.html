Fyzické vs. logické vlastnosti

Živá ukázka — fyzické vs. logické vlastnosti

inline-size·block-size

Tlačítko přepíná směr čtení (dir) na celém demo kontejneru. Logická verze se automaticky překlopí do RTL, fyzická ne.

Aktuální směr: LTR
Fyzické vlastnosti
physical

Název položky

Tato karta používá margin-left, padding-top a border-left. Při přepnutí do RTL se tyto vlastnosti nepřeklopí — levý okraj zůstane vlevo.

fyzická nepřeklápí se 
/* Fyzické vlastnosti */
.card {
  margin-left: 1rem;
  padding-top: 1.5rem;
  border-left: 3px solid;
  text-align: left;
}
Logické vlastnosti
logical

Název položky

Tato karta používá margin-inline-start, padding-block-start a border-inline-start. Při přepnutí do RTL se automaticky překlopí na správnou stranu.

logická překlopí se ✓ 
/* Logické vlastnosti */
.card {
  margin-inline-start: 1rem;
  padding-block-start: 1.5rem;
  border-inline-start: 3px solid;
  text-align: start;
}

Logical Properties

/* Fyzické vlastnosti jsou vázané na viewport (top/right/bottom/left) */
/* Logické vlastnosti jsou vázané na směr psaní (block / inline osy) */

/* Inline osa → v LTR = vodorovná (←→), v RTL se otočí */
/* Block osa  ↓ v LTR = svislá (↑↓), nemění se */

Proč záleží?

/* RTL jazyky (arabština, hebrejština) nebo vertical writing (japonština) —
   logické vlastnosti se automaticky přizpůsobí bez změny CSS.
   Fyzické vlastnosti zůstanou vždy na stejné straně viewportu.        */
