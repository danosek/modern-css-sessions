Nesting bez &

Nesting bez & — element selektory uvnitř bloku ampersand nepotřebují:

nesting

Základní karta

Element h2 a p jsou psané bez ampersandu — stačí je vnořit přímo. Prohlédni si hover efekt.

footer · h2 a p bez &

Aktivní stav

Modifier &.no-amp-card--active stále vyžaduje &. Uvnitř pak opět h2 bez &.

footer · &.--active + h2 bez & 
.no-amp-card {
  border-radius: var(--radius-strong);
  padding: var(--base-3) var(--base-4);

  /* element selektory nepotřebují & — platí od Chrome 112+, FF 117+, Safari 17.2+ */
  h2 { font-size: var(--font-size-xl); color: var(--text-primary); }
  p  { color: var(--text-secondary); line-height: 1.6; }
  span { font-family: var(--font-stack-monospace); }

  /* třídy, pseudo-třídy a pseudo-elementy & vyžadují nadále */
  &:hover          { box-shadow: var(--elevation); }
  &.no-amp-card--active {
    border-color: var(--surface-brand-primary-strong);
    h2 { color: var(--text-brand-primary); }  /* opět bez & */
  }
}

Pravidlo: selektory začínající identifikátorem (element, *) implicitně dostanou & jako předponu — výsledek je .no-amp-card h2. Třídy (.foo), atributy ([type]) a pseudo-třídy (:hover) se bez & nepřipojí správně — prohlížeč je interpretuje jako & .foo (s mezerou = potomek), ne &.foo (bez mezery = modifier).
